# BizAgentOS PRD（产品需求文档）

> 文档类型：产品需求文档  
> 版本：v2.0（融合前端实现版）  
> 日期：2026-04-20  
> 说明：本文档基于 v1.9 产品定义，整合当前前端已实现能力，形成完整的产品需求规范

## 1. 产品定义

### 1.1 高层摘要

BizAgentOS 是一套企业级系统，服务于人类员工和 AI 员工，构建在一个共享内核之上，以三类智能体构成完整产品体系，并根据用户层级呈现不同入口。

- BizAgentOS 是组织智能的核心操作系统，不是单一聊天助手，也不是试图替代所有知识基础设施的大一统底座。
- 它通过数字员工、管理智能体和企业效能管理智能体，把知识工程、数据治理、流程梳理、分析和执行组织成持续运行的生产系统。
- 它支持企业把已有知识处理、组织、结构化并转化成适合 AI 使用的知识与资料；本体、知识图谱、检索资产、规则语义、SOP 结构化和可复用知识包，都是这一知识工程过程中的可能成果。
- 它可以按需把这些成果发布到外部知识库、图数据库或其他企业系统，并让这些资产持续服务于治理、分析和业务执行。

### 1.2 三智能体模型

#### 1.2.1 数字员工

- 知识工程智能体
- 数据治理智能体
- 智能数据分析智能体
- 作为被管理、被委派、被教学和被观测的执行单元存在
- 同一类数字员工不是只有一个，可以按工作线拥有多个实例
- 可以被单独打开为聚焦视图，但该聚焦视图应对应单个实例，而不是整条工作线
- 不作为普通人类员工的顶层主入口

#### 1.2.2 管理智能体

- 每位人类员工配备一个个人管理智能体
- 是普通人类员工默认使用和查看的主入口
- 管理、编排、培训和审核多个数字员工实例
- 支持以自然语言表达管理目标、约束、优先级和背景，再转换成结构化委派、审核或教学草案
- 在委派前判断是否存在合适的数字员工与 Skill 覆盖
- 在 Skill 覆盖不足时，转入能力教学和 Skill 形成流程
- 审核任务结果，并决定是否把成果沉淀为可复用 Skill、知识资产或工作方法
- 内含 Skills 复用中枢，用于组织团队可复用能力
- 可进入单个数字员工实例的聚焦视角，查看任务、产出、质量和挂载 Skill 状态

#### 1.2.3 企业效能管理智能体

- 供企业管理层使用
- 是高于管理智能体的一层组织级智能体
- 聚焦组织级效能、管理者维度产能、风险和运营洞察
- 默认关注某个管理者及其数字员工组合的总体产出，而不是只盯单个数字员工
- 在需要时可兼有部分跨数字员工的调度和管理能力
- 当前阶段保留为独立产品定义，但不阻塞管理智能体主闭环

### 1.3 Agent Skills 的产品定义

- Agent Skills 是 BizAgentOS 中的一等资产
- 它不是随手拼接的一段 prompt，而是把 SOP、规则、方法、流程、工具使用方式、判断边界和成功标准，固化成 Agent 可可靠调用的执行能力
- Skill 既可以服务某个数字员工实例，也可以被挂载到某条数字工作线，供多个实例复用
- 管理智能体需要负责 Skill 的教学、审核、挂载、复用和淘汰，而不只是"发任务"

### 1.4 管理智能体中的 Skills 复用中枢

- 管理智能体中必须存在一个 `Skills 复用中枢`
- 它用来把一次任务里验证过的有效做法，升级成团队可复用的 Skill
- 它既服务"能力教学"形成的新 Skill，也服务"任务结果审核"后沉淀出来的 Skill
- 它至少要支持：查看 Skill 目录、查看来源、查看适用范围、挂载到工作线或实例、升级版本、停用

### 1.5 任务委派与能力教学的关系

- 管理智能体不应只提供一个"把任务指给谁"的分派动作
- 管理智能体应先判断：是否存在合适的工作线、是否存在合适的实例、该实例是否具备足够的 Skill 覆盖
- 如果没有足够 Skill 覆盖，正确路径不是盲派，而是先进入能力教学
- 任务完成后，管理智能体还应判断：这次任务里是否形成了值得沉淀的新 Skill

### 1.6 本体、知识图谱与知识工程的产品定位

- BizAgentOS 的价值不在于把本体、知识图谱或图数据库全部内嵌进自身
- BizAgentOS 的价值首先在于把企业已有知识处理、组织、结构化并转化成适合 AI 使用的知识与资料体系
- 本体、知识图谱、图友好结构、检索资产、规则语义、SOP 结构化结果和可复用知识包，都是知识工程过程中的可能成果，而不是唯一成果
- 当业务需要更强的关系表达、图推理或外部共享时，本体与知识图谱可由知识工程智能体及其技能构建，并发布到外部图数据库、知识系统
- BizAgentOS 自身承担编排、治理、审计、复用和持续演化的核心操作系统角色

### 1.7 BizAgentOS 不是什么

- 不是企图覆盖所有企业工作的单一聊天助手
- 不是在不同角色名称下共享的仪表板
- 不是由三个平权主入口组成的 launcher 产品
- 不是把数字员工当作与管理智能体平级的普通用户前端
- 不是以治理控制台作为第三角色的主要产品形态
- 不是试图替代企业现有知识库、图数据库和所有数据基础设施的"大一统底座"
- 不是自主企业模拟系统

### 1.8 核心产品原则

共享内核 **不** 意味着共享界面。

内核是共享的。产品入口层级不是。

- 普通人类员工默认从管理智能体进入系统
- 数字员工通过管理智能体被管理，并以聚焦视图被查看
- 企业效能管理智能体面向更高层管理者，独立于管理智能体但可复用部分管理能力

## 2. 核心用户问题

### 2.1 数字员工面临的问题

企业需要能够执行实际工作的 AI 员工，而非仅能回答问题。

初始目标领域：

- **知识工程**：将散落的人类可读文档、制度、规则、SOP 和业务语义转化为结构化知识资产，供智能体和 LLM 使用
- **数据治理**：提升经典治理工作，如规则、权属、质量、定义和整改
- **智能数据分析**：利用已治理的数据和结构化知识来回答业务问题并生成分析

### 2.2 管理面临的问题

每位人类员工需要一种方式来管理多个数字员工，而非使用孤立的 AI 工具。

他们需要：

- 分配工作
- 用自然语言描述任务目标、约束、背景和优先级，并让系统生成可确认的结构化草案
- 判断某个任务是否存在合适的数字员工与 Skill 覆盖
- 在缺少 Skill 覆盖时，转入能力教学，而不是盲目委派
- 专注查看单个数字员工实例的当前状态、产出、质量信号和已挂载 Skill
- 跨多个数字员工实例拆分工作
- 教授新技能
- 将标准作业流程（SOP）和工作方法转化为可复用 Skill
- 审核产出，并把高质量做法升级为团队可复用能力

### 2.3 企业效能面临的问题

企业管理层需要一个更高层次的智能层，展示：

- 工作在哪些环节减速
- 哪些管理者及其数字员工组合真正在交付价值
- 哪些团队的数字工作与人工工作协作失衡
- 下一步应采取哪些运营行动

## 3. 典型落地场景

### 3.1 企业制度与 SOP 的知识工程化

- 业务团队上传制度、流程文档、口径说明、模板和历史材料
- 知识工程智能体提取概念、术语、规则、流程节点和关键关系，生成适合 AI 使用的结构化知识资产
- 这些成果可表现为检索资产、规则语义、SOP 结构化结果、知识包，必要时也可进一步形成本体或知识图谱
- 管理智能体审核和复用这些成果，使后续数字员工不再从零理解业务

### 3.2 跨部门知识整合与外部知识系统发布

- 当多个部门对同一对象存在不同定义、流程或口径时，管理智能体组织知识工程智能体和数据治理智能体协同处理
- 数字员工将分散知识整理为统一对象模型、关系描述、规则边界和结构化资产
- 当企业需要图谱化沉淀或与现有平台打通时，相关技能可将成果发布到外部知识库、图数据库或其他企业系统
- BizAgentOS 负责委派、审核、追踪、审计和复用，而不是替代这些外部系统

### 3.3 结构化知识驱动分析与业务执行

- 智能数据分析智能体基于已治理数据和知识工程产出的结构化资产理解业务问题
- 它可以调用规则表达、概念定义、SOP 结构和关系语义，生成更贴近业务上下文的分析、建议和解释
- 流程执行型数字员工也可以复用同样的知识资产，在客服、运营、法务或供应链流程中完成判断与推进
- 企业效能管理智能体进一步观察这些资产是否真正带来自动化覆盖、效率提升和风险下降

## 4. 产品界面与入口层级

### 4.1 当前阶段必须提供的界面


| 路由                                                       | 界面       | 说明                |
| -------------------------------------------------------- | -------- | ----------------- |
| `/workspace`                                             | 管理智能体主界面 | 普通人类员工默认入口        |
| `/workspace/manager-agent`                               | 管理智能体详情  | 任务委派、跟踪、审核        |
| `/workspace/manager-agent/digital-employees/:employeeId` | 数字员工聚焦视图 | 单实例任务、产出、质量、Skill |
| `/workspace/digital-employees`                           | 数字员工广场   | 兼容迁移期重定向路径，非顶层主入口 |


### 4.2 后续阶段保留的界面


| 路由                            | 界面        | 说明      |
| ----------------------------- | --------- | ------- |
| `/workspace/efficiency-agent` | 企业效能管理智能体 | 面向高层管理者 |


### 4.3 界面差异化要求

- 管理智能体与企业效能管理智能体必须是两套完整且独立的前端 UI，不得仅在同一壳层中替换文案或卡片
- 数字员工聚焦视图从属于管理智能体，但允许保留独立的运行面板视觉，用于强调单员工运行状态与管理动作
- 管理智能体首页第一屏必须优先展示：**待处理事项**、**委派决策与 Skill 覆盖判断**、**执行跟踪**

## 6. 管理智能体主界面（首页）

### 6.1 页面定位

管理智能体是普通人类员工的**默认主入口**，不是传统的 Dashboard，而是"管理操作的起点"。

第一屏必须优先展示：**待处理事项**、**委派决策与 Skill 覆盖判断**、**执行跟踪**。

### 6.2 布局比例

- **左侧（75%）**：管理操作面板（任务委派、执行跟踪、审核队列）
- **右侧（25%）**：管理助手聊天（自然语言交互层）

### 6.3 右侧：管理助手聊天

#### 6.3.1 聊天头部

- 图标：👑 皇冠
- 标题：管理智能体
- 副标题：个人管理助手 · 任务委派与审核
- 标签：「委派模式」+ 在线状态指示器

#### 6.3.2 对话内容

**预设对话流**：

1. 系统消息："管理智能体已就绪"
2. 助手欢迎语：介绍能力范围（任务委派、Skill 覆盖检查、执行跟踪、结果审核、能力教学）
3. 用户输入管理目标
4. 助手返回结构化草案（待用户确认）

**核心交互模式**：

- 自然语言输入 → 生成结构化委派草案 → 用户在结构化界面中确认/修改 → 执行
- 不是纯聊天，而是"自然语言增强的控制面"

#### 6.3.3 输入框

- 占位文案："描述任务目标、约束和优先级..."
- 支持 `Enter` 发送，`Shift+Enter` 换行
- @唤起：快速选择数字员工实例或 Skill

### 6.4 左侧：管理操作面板

#### 6.4.1 待处理事项区

- 待审核任务列表
- Skill 覆盖不足告警（需转入能力教学）
- 执行阻塞提醒

#### 6.4.2 委派决策区

- 任务输入框（自然语言）
- 候选工作线/实例建议
- Skill 覆盖检查结果
  - ✅ 覆盖充足：显示可委派按钮
  - ⚠️ 覆盖不足：显示「先教学再委派」按钮

#### 6.4.3 执行跟踪区

- 已委派任务的进度看板
- 每个任务卡片：任务名、执行实例、当前阶段、阻塞状态、预计完成时间
- 点击任务进入工作线群聊详情

#### 6.4.4 Skills 复用中枢入口

- Skill 目录快捷查看
- 最近沉淀的 Skill
- 待审核的 Skill 草案

## 7. 数字员工广场（Agent 页）

### 7.1 页面定位

数字员工广场是**企业数字员工资产的统一管理中心**，从管理智能体可进入，但**不是平级主入口**。

这里展示所有可被管理的数字员工实例，支持选中后进入聚焦视图。

### 7.2 布局逻辑

- **默认状态**：数字员工广场独占中间区域
- **选中实例后**：左侧列表占 75%，右侧聚焦视图占 25%

### 7.3 左侧：数字员工广场

#### 7.3.1 页面头部

- 标题：数字员工广场
- 副标题：查看和管理所有数字员工实例，点击卡片进入聚焦视图
- 「接入工作线」按钮

#### 7.3.2 筛选标签

- 全部 / 知识工程 / 数据治理 / 智能分析
- 点击切换高亮

#### 7.3.3 数字员工实例卡片网格（3 列）

**卡片内容**：

- 顶部行：Avatar（emoji）+ 实例名称 + 状态图标（在线/忙碌/离线）+ 工作线标签
- 描述文本（2 行截断）
- 已挂载 Skill 标签列表
- 底部行：累计调用次数 + 聚焦按钮（hover 显示）+ 设置按钮

**交互**：

- 点击卡片：选中该实例，右侧滑出聚焦视图
- 选中态：边框变为主题色，带 ring 阴影
- Hover：显示「聚焦视图」按钮

### 7.4 右侧：数字员工聚焦视图

#### 7.4.1 面板头部

- 实例 Avatar + 名称 + 所属工作线 + 在线状态 + 累计调用次数
- 关闭按钮

#### 7.4.2 当前任务区

- 正在执行的任务名称和进度
- 当前阶段和阻塞状态

#### 7.4.3 已挂载 Skill 区

- 显示该实例已挂载的所有 Skill
- 支持快速解除挂载

#### 7.4.4 历史产出区

- 最近完成的任务列表
- 每个任务：名称、完成时间、质量评分

#### 7.4.5 单聊测试区（可选折叠）

- 与该实例的直接对话测试
- 用于教学、调试或单任务沟通

## 8. Skills 复用中枢（Skill 页）

### 8.1 页面定位

Skills 复用中枢是管理智能体的核心能力之一，负责：

- 查看 Skill 目录
- 查看 Skill 来源和适用范围
- 挂载到工作线或实例
- 升级版本、停用
- 从任务成果中沉淀新 Skill

### 8.2 布局逻辑

- **默认状态**：Skill 列表独占中间区域
- **选中 Skill 后**：左侧列表占 75%，右侧详情/交互面板占 25%

### 8.3 左侧：Skill 目录

#### 8.3.1 页面头部

- 标题：Skills 复用中枢
- 副标题：管理团队可复用能力，共 X 个 Skill
- 「从任务沉淀 Skill」按钮

#### 8.3.2 统计卡片（4 列）

- 已启用 / 已停用 / 来源任务数 / 适用工作线数

## 10. 主要工作流

### 10.1 任务委派闭环（管理智能体核心）

```
1. 管理智能体接收自然语言任务描述
   ↓
2. 生成结构化委派草案（目标、约束、优先级）
   ↓
3. 判断候选工作线和实例
   ↓
4. 检查 Skill 覆盖
   ├── 覆盖充足 → 进入步骤 5
   └── 覆盖不足 → 转入能力教学（形成 Skill 草案 → 挂载 → 再委派）
   ↓
5. 用户确认委派
   ↓
6. 数字员工实例执行
   ↓
7. 执行跟踪（进度、阻塞、阶段产出）
   ↓
8. 结果审核
   ├── 通过 → 判断是否沉淀为 Skill
   └── 不通过 → 退回修改或终止
```

### 10.2 能力教学工作流

```
1. 管理智能体判断 Skill 覆盖不足
   ↓
2. 进入教学模式
   ↓
3. 输入：SOP、规则、方法、样例、判断边界
   ↓
4. 系统生成 Skill 草案
   ↓
5. 测试验证
   ↓
6. 审核通过 → 挂载到实例/工作线
   ↓
7. 转入正常委派流程
```

### 10.3 Skill 沉淀工作流

```
1. 任务完成，结果审核通过
   ↓
2. 管理智能体判断：是否有值得沉淀的新做法？
   ↓
3. 提取通用方法、规则、判断边界
   ↓
4. 生成 Skill 草案
   ↓
5. 录入 Skills 复用中枢
   ↓
6. 标记适用范围和来源
```

## 11. 版本与阶段规划

### 11.1 当前 `v0.1.0` 小型可见 MVP

`v0.1.0` 不追求做完整产品，而是只验证以下最小闭环：

- 一个真实的管理智能体主界面，作为普通人类员工默认入口
- 一条知识工程工作线
- `1-2` 个可被管理的数字员工实例
- 一套"任务输入 -> 候选实例建议 -> Skill 覆盖检查 -> 委派 / 教学分流 -> 执行跟踪 -> 结果审核"的闭环
- 一个可从管理智能体点开的单实例聚焦视图
- 一个稳定的底层内核

`v0.1.0` 明确不要求：

- 同时完成三条工作线的完整产品化
- 完整的企业效能管理智能体产品闭环
- 完整的 Skills 复用中枢与 Skill 生命周期管理
- 完整的持久化、租户隔离、sandbox、registry、observability

### 11.2 `v0.1.1` Skill 闭环补强

- Skill 沉淀决策
- Skills 复用中枢
- 能力教学输入
- Skill 草案、挂载、解除挂载与停用

### 11.3 `v0.1.2` 多工作线扩展

- 数据治理工作线
- 智能分析工作线

### 11.4 `v0.1.3` 管理强化

- 管理者维度的产能汇总
- 跨实例协调
- 知识资产发布记录与外部连接
- 管理智能体自然语言交互层

### 11.5 `v0.2.0` 高层与基础设施

- 企业效能管理智能体
- 持久化与租户隔离
- Sandbox 验证运行时
- Registry 与模板
- 部署与可观测性
- 多实例并行委派与协同任务编排

## 12. 验收标准

### 12.1 `v0.1.0` MVP 验收标准

1. 三智能体模型在文档和界面中保持一致
2. 管理智能体是普通人类员工的默认主入口
3. 单个数字员工实例可以从管理智能体进入聚焦视图，但数字员工不作为平级顶层主入口出现
4. 同一类数字员工可拥有多个实例，且界面与文档不再混淆"工作线"和"实例"
5. 管理智能体能在委派前完成候选 Agent 建议和 Skill 覆盖检查
6. 当 Skill 覆盖不足时，管理智能体可转入能力教学，而不是只能硬性分派
7. 管理智能体主界面可以持续展示委派任务的执行进度、阻塞、阶段输出和相关详情
8. 数字员工实例聚焦视图可以展示任务、输出、质量信号、证据链与已挂载 Skill
9. 管理智能体可以审核任务结果，并把合适成果沉淀到 Skills 复用中枢
10. 护栏保持可用，但不替代产品叙事
11. 知识工程智能体能够驱动知识资产构建链路，并可按需形成本体 / 知识图谱、发布外部系统并服务下游智能体
12. 企业效能管理智能体的产品定义保留，但其完整闭环不阻塞当前阶段验收

# Agent-OS 图谱可视化 UI 规范

> 可直接指导前端开发的图谱绘制方案

---

## 一、项目图谱：怎么画

### 1.1 整体布局（力导向 + 同心圆分层）

```
                         [⚠️ 风险节点]
                              │
        [Agent-1] ──── [🏢 项目中心] ──── [Agent-2]
           │                  │                  │
       [Skill-A]          [📁 资产]         [Skill-B]
           │                                    │
       [Skill-C]                           [产出-1]

        [Agent-3] ──── [🏢 项目2] ──── [Agent-4]
```

**布局规则**：
- 项目节点固定在中心（layer 0），半径 40px
- Agent 节点环绕项目（layer 1），距离中心 120-180px，半径 24px
- Skill 节点挂在 Agent 外侧（layer 2），距离 Agent 60px，半径 16px
- 资产节点在项目下方（layer 2），半径 14px
- 风险节点在项目边缘随机浮现，半径 12px

### 1.2 节点样式详解

#### 项目节点（中心大圆）

```
┌─────────────────────────────┐
│                             │
│      ┌─────────────┐        │
│      │   供应链金融  │        │  ← 项目名称，14px 粗体
│      │   风控平台    │        │
│      │   ┌──────┐   │        │
│      │   │  68%  │   │        │  ← 进度环，SVG circle
│      │   └──────┘   │        │     stroke-dasharray 动态
│      │   健康状态    │        │  ← 状态标签
│      └─────────────┘        │
│                             │
└─────────────────────────────┘
```

**CSS/SVG 参数**：
```
尺寸：80x80px（半径 40）
背景：渐变 #1e3a5f → #3b5998（深蓝）
边框：2px solid rgba(255,255,255,0.3)
阴影：0 8px 32px rgba(30,58,95,0.25)

进度环：
  - 外圈：r=34，stroke=#e17055（珊瑚色），stroke-width=3
  - 进度：stroke-dasharray = 2πr × (进度%), 动画 1s ease-out
  
状态标签：
  - 健康：绿色圆点 + "正常"
  - 警告：黄色圆点 + "2项风险"  
  - 危险：红色脉动圆点 + "已阻塞"
```

#### Agent 节点（环绕中圆）

```
    ┌─────────┐
    │  🤖     │  ← Agent 图标（lucide Bot）
    │ 财报分析 │  ← 名称，11px
    │ ● 忙碌  │  ← 状态指示器
    │ ████░░░ │  ← 负载条（微型）
    └─────────┘
```

**CSS 参数**：
```
尺寸：48x48px（半径 24）
背景：
  - 空闲：#0d9488（teal）
  - 忙碌：#e17055（珊瑚）脉动
  - 异常：#dc2626（红色）闪烁
  
状态圆点：
  - 右上角 8px 圆点
  - 空闲：绿色 #22c55e
  - 忙碌：橙色 #f97316（animate-pulse）
  - 异常：红色 #ef4444（animate-pulse-dot）

负载条：
  - 宽度 36px，高度 3px，圆角
  - 填充色根据负载率渐变：绿 → 黄 → 红
```

#### Skill 节点（小圆/六边形）

```
   ┌────┐
  / 图  \   ← 六边形，16px
 │ 表生成 │   ← 名称隐藏，悬停显示
  \      /
   └────┘
```

**CSS 参数**：
```
形状：六边形（SVG polygon 或 clip-path）
尺寸：32x32px（半径 16）
背景：#7c3aed（紫色）半透明 0.85
边框：1px solid rgba(255,255,255,0.2)
悬停：放大 1.3 倍 + tooltip 显示名称和调用次数
```

#### 风险节点（动态浮现）

```
    ⚠️
   阻塞
   任务
```

**CSS 参数**：
```
尺寸：24x24px
动画：从项目边缘向外扩散的涟漪
      @keyframes risk-ripple {
        0% { transform: scale(0.8); opacity: 0; }
        50% { opacity: 1; }
        100% { transform: scale(1.5); opacity: 0; }
      }
颜色：#dc2626（红色）
触发：当任务逾期 / Agent 失败 / 审批超时时自动浮现
```

### 1.3 边的绘制

```
归属边（项目 → Agent）：
  stroke: rgba(100,100,100,0.3)
  stroke-width: 1.5
  stroke-dasharray: none

挂载边（Agent → Skill）：
  stroke: rgba(124,58,237,0.4)  ← 紫色虚线
  stroke-width: 1
  stroke-dasharray: 4,4

依赖边（项目A → 项目B）：
  stroke: #e17055
  stroke-width: 2
  marker-end: url(#arrowhead)
  动画：如果阻塞，虚线流动动画
```

## 二、任务图谱：怎么画

### 2.1 布局选择

**推荐：横向泳道流程图（GitHub Actions 风格）**

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  🎯 用户  │───→│ 🧠 群主  │───→│ 🤖 Agent │───→│ 👤 审批  │───→│ 📦 产物  │
│  目标输入  │    │  任务拆解  │    │  财报分析  │    │  是否通过  │    │  图表产出  │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
                                      │
                                      ↓
                               ┌─────────┐
                               │ 🛠️ Skill │
                               │ 数据提取  │
                               └─────────┘
```

**布局规则**：
- 横向排列，每个步骤一个卡片
- 卡片宽度 160px，高度 80px，圆角 12px
- 步骤间距 40px
- 审批节点用菱形（旋转 45° 的正方形）
- 失败路径向下分叉，用红色虚线

### 2.2 节点卡片样式

#### 标准执行步骤（矩形卡片）

```
┌─────────────────────┐
│ 🤖 财报分析 Agent    │  ← 图标 + Agent 名称，12px 粗体
│                     │
│ 开始：10:02          │  ← 元数据，10px 灰色
│ 耗时：1m 23s         │
│                     │
│ [████████░░] 80%    │  ← 进度条（执行中显示）
│                     │
│ 状态：✅ 成功        │  ← 状态标签
└─────────────────────┘
```

**CSS 参数**：
```
宽度：160px
高度：80px（紧凑）/ 120px（展开）
背景：白色，rgba(255,255,255,0.9)
边框：1px solid #e5e5e5
圆角：12px
阴影：0 2px 8px rgba(0,0,0,0.06)

状态色：
  - 排队中：灰色 #9ca3af，左侧 3px 边框
  - 执行中：蓝色 #3b82f6，左侧边框 + 顶部脉动条
  - 成功：绿色 #22c55e，左侧边框
  - 失败：红色 #ef4444，左侧边框 + 卡片背景淡红
  - 降级：橙色 #f97316，左侧边框
```

#### 审批节点（菱形）

```
        ┌────┐
       / 👤  \
      /  审批  \
     /  待处理   \
     \          /
      \  2h   /
       └────┘
```

**CSS 参数**：
```
形状：菱形（transform: rotate(45deg)）
尺寸：72x72px
背景：#fef3c7（淡黄色）
边框：2px solid #f59e0b（琥珀色）
动画：待审批时边框脉动（pulse-border）

内部文字（反向旋转 -45deg）：
  - "审批" 标签
  - "已通过" / "已驳回" / "待处理"
  - 等待时长
```

#### 产物节点（文件卡片）

```
┌─────────────┐
│ 📊 Q3营收对比 │  ← 产物名称
│    柱状图    │
│             │
│ [预览缩略图] │  ← 小型图表预览
│             │
│ 📥 下载     │  ← 操作按钮
└─────────────┘
```

**CSS 参数**：
```
宽度：140px
高度：auto（根据内容）
背景：白色
边框：1px solid #e5e5e5
圆角：8px
悬停：阴影加深 + 放大 1.02 倍
```

### 2.3 失败与降级路径

```
          正常路径
┌────────┐    ┌────────┐
│ 步骤 A │───→│ 步骤 B │
└────────┘    └────────┘
     │
     │ 失败
     ↓
┌────────┐    ┌────────┐
│ ⚠️ 异常 │───→│ 🔄 降级 │  ← 红色虚线分叉
│ 数据缺失 │    │ 使用缓存 │
└────────┘    └────────┘
     │
     │ 人工介入
     ↓
┌────────┐
│ 👤 处理 │
└────────┘
```

**CSS 参数**：
```
异常节点：
  背景：#fef2f2（淡红）
  边框：1px dashed #ef4444
  图标：⚠️ + 错误简述
  按钮："重试" / "跳过" / "人工处理"

降级边：
  stroke: #f97316
  stroke-dasharray: 6,3
  动画：虚线流动（marquee）
```

## 三、联动交互设计

### 3.1 项目图谱 → 任务图谱

```
用户行为：点击项目图谱中的 Agent 节点
系统响应：
  1. 右侧展开「该 Agent 当前任务列表」
  2. 点击某个任务 → 切换到任务图谱（Flow View）
  3. 任务图谱中该 Agent 的节点高亮闪烁
```

### 3.2 任务图谱 → 项目图谱

```
用户行为：点击任务图谱中的产物节点
系统响应：
  1. 右侧预览产物内容
  2. 产物节点显示「所属项目」标签
  3. 点击标签 → 切换到项目图谱
  4. 项目图谱中对应资产节点高亮
```

### 3.3 风险联动

```
任务图谱中发生异常（步骤失败）
  ↓
自动在项目图谱中生成/更新风险节点
  ↓
风险节点脉动动画提醒
  ↓
点击风险节点 → 定位到任务图谱中的异常步骤
```

## 四、颜色编码系统

| 类型 | 颜色 | 用途 |
|------|------|------|
| 项目 | #1e3a5f 深蓝 | 中心节点，稳重 |
| Agent | #0d9488 teal | 执行者，活跃 |
| Skill | #7c3aed 紫色 | 能力单元，工具 |
| 资产 | #475569 灰 | 产出物，结果 |
| 洞察 | #d97706 琥珀 | 智能发现，警示 |
| 风险 | #dc2626 红色 | 异常，需关注 |
| 人类 | #2563eb 蓝色 | 人工节点，审批 |
| 成功 | #22c55e 绿色 | 完成状态 |
| 进行中 | #3b82f6 蓝色 | 执行状态 |
| 等待 | #9ca3af 灰色 | 排队状态 |

## 五、动画规范

| 场景 | 动画 | 参数 |
|------|------|------|
| Agent 忙碌 | 节点脉动 | scale 1.0→1.05→1.0, 2s infinite |
| 风险告警 | 涟漪扩散 | scale 0.8→1.5, opacity 1→0, 1.5s |
| 步骤执行中 | 顶部进度条 | width 0→100%, 渐变流动 |
| 审批等待 | 边框闪烁 | border-color 淡→深→淡, 1.5s |
| 异常出现 | 卡片抖动 | translateX ±3px, 0.3s |
| 产物生成 | 飞入动画 | 从 Agent 飞向产物区, 0.5s ease-out |
| 图谱切换 | 淡入淡出 | opacity 0→1, 0.3s |

## 六、开发优先级建议

| 优先级 | 内容 | 原因 |
|--------|------|------|
| P0 | 项目图谱：项目+Agent+Skill 基础节点 | 用户每天第一眼看到的 |
| P0 | 任务图谱：横向流程图标准步骤 | 建立 Agent 可信度 |
| P1 | 状态颜色编码 + 脉动动画 | 让用户感知"活着" |
| P1 | 风险节点自动浮现 | B 端最怕不知道出问题了 |
| P2 | 审批节点菱形 + 闪烁 | 人机协作的核心交互 |
| P2 | 产物节点可点击预览 | 让用户感知 Agent 价值 |
| P3 | 失败降级路径可视化 | 异常处理的高级能力 |
| P3 | 两图谱联动跳转 | 完整体验闭环 |
